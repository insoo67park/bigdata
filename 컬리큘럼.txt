- 파일 시스템의 단점

  -> 데이터 접근의 어려움 (각 테스트 수행을 위해 각기 새로운 프로그램이 필요)
  -> 데이터 아이솔레이션
  -> 무결성 문제 (데이터 추가, 삭제 등의 어려움) -> 동시성 문제
  -> DBMS로 문제점 해결!!!
  Level of Abstraction
  - Physical Level: 레코드가 어떻게 저장되는지 기술함
  - Logical Level: 데이터베이스에 저장된 데이터와 데이터간 관계를 기술함
  - View Level: Application programs hide details of data types. Views can also hide information (such as an employee's salary) for security purpose
  
  Schema: the logical structure of the database (physical schema, logical schema)
  Instance: the actual contents of the database at a particular point in time 
  physical data independence: the ability to modify the physical schema without changing the logical schema
   -> application depends on the logical schema
   -> in general the interface between the various levels and components should be well defined so that changes in some parts do not seriously influence others.
